# Manual Test Specification

## 目的
このテスト仕様書は、Umeda Survival Map Web アプリをモバイル利用想定で手動検証するための手順書です。
Web 開発の経験が少ない人でも、順番どおりに実施できるように記載しています。

## 前提条件
- アプリはローカル開発サーバーで起動しておくこと
  - `npm run dev -- --port 3001` で起動
  - ブラウザで `http://localhost:3001` にアクセス
- スマートフォン実機または、PC のブラウザでモバイルエミュレーションを使うこと
- テスト実行前にブラウザのキャッシュをクリアするか、プライベート/シークレットウィンドウを使うこと

## テスト環境
- OS: Windows / macOS / Android / iOS など
- ブラウザ: Chrome / Edge / Safari いずれか
- 画面サイズ: スマホ縦向き（例: 390x844）
- URL: `http://localhost:3001`

## 検証項目一覧
1. アプリ起動および表示確認
2. モバイルレイアウト確認
3. 位置情報許可フロー
4. 位置情報拒否フロー
5. カテゴリフィルタ動作
6. スポット詳細表示
7. スポット投稿フォーム動作
8. localStorage 保存・再表示
9. PWA / マニフェスト確認

---

## 1. アプリ起動および表示確認
### 手順
1. ブラウザで `http://localhost:3001` を開く
2. 画面が読み込まれるのを待つ

### 想定結果
- 画面上部に `Japan Survival Map` と `Umeda Essentials` が表示される
- `Mobile web PWA` のラベルが見える
- `Add new spot` ボタンが表示される
- 地図エリアが表示される

### 合否判定
- 表示されない項目がある場合は `FAIL`
- すべて正常に見える場合は `PASS`

---

## 2. モバイルレイアウト確認
### 手順
1. スマホ実機またはモバイルエミュレーションで画面を表示
2. 画面をスクロールして、全体レイアウトを確認する
3. フィルタ・ボタン・一覧カードが重なっていないかを確認する

### 想定結果
- ヘッダー・説明文・ボタンが縦に並び、左右に見切れない
- フィルタエリアが画面幅に収まる
- マップが適切な高さで表示される
- 一覧カードが画面幅いっぱいに広がり、タップできる

### 合否判定
- レイアウト崩れや文字切れがあれば `FAIL`
- 画面が読みやすく操作しやすければ `PASS`

---

## 3. 位置情報許可フロー
### 手順
1. ブラウザが位置情報の許可を求めた場合は「許可する」を選択
2. 画面内の表示を確認

### 想定結果
- `Location enabled. Showing nearby spots.` のメッセージが表示される
- 地図上に現在地マーカーが表示される（青やピンなど）
- スポット一覧に距離が表示される

### 合否判定
- メッセージが出ない、またはマーカーが表示されない場合は `FAIL`
- 表示が正常なら `PASS`

---

## 4. 位置情報拒否フロー
### 手順
1. ブラウザの位置情報許可ダイアログで「拒否」または「許可しない」を選択
2. 画面の表示を確認

### 想定結果
- `Location denied or unavailable. Showing Umeda center.` のメッセージが表示される
- アプリがクラッシュせず、地図は Umeda 中心で開く
- スポット一覧は表示されたまま動作する

### 合否判定
- アプリがエラーで停止した場合は `FAIL`
- メッセージが正しく出ていれば `PASS`

---

## 5. カテゴリフィルタ動作
### 手順
1. フィルタ一覧から `Toilet` を選択する
2. `ATM` や `Coin locker` など別カテゴリも順に選択する

### 想定結果
- 選択したカテゴリに一致するスポットのみ一覧に残る
- 別カテゴリを選ぶと一覧が切り替わる
- 画面上の選択状態が変化する（ボタンの強調など）

### 合否判定
- フィルタが切り替わらない場合は `FAIL`
- 期待どおり絞り込まれれば `PASS`

---

## 6. スポット詳細表示
### 手順
1. スポット一覧のカードをタップする
2. 下部の詳細シートが開くか確認する

### 想定結果
- `Spot details` の見出しまたは詳細モーダルが表示される
- スポット名・カテゴリー・説明・距離などが見える
- 閉じるボタンや外側タップで閉じる場合は、正常に閉じる

### 合否判定
- 詳細が開かない場合は `FAIL`
- 正常に詳細が表示されれば `PASS`

---

## 7. スポット投稿フォーム動作
### 手順
1. `Add new spot` ボタンをタップする
2. フォームが表示されたら以下を入力する
   - Name: `Test Local ATM`
   - Category: `ATM`
   - Description: `Local ATM near the station.`
   - Tourist note: `Open 24 hours.`
   - Tags: `test, local`
   - Hours: `24 hours`
   - Payment note: `Free`
3. `Save spot` ボタンをタップする

### 想定結果
- フォームが閉じる
- `Test Local ATM` のカードがスポット一覧に追加される
- 追加後に `ATM` フィルタを選んでいれば、そのフィルタ結果に表示される

### 合否判定
- フォーム入力後に新しいスポットが出ない場合は `FAIL`
- 追加結果が確認できれば `PASS`

---

## 8. localStorage 保存・再表示
### 手順
1. 上記の投稿フォームでスポットを追加する
2. ブラウザをリロードする
3. 画面を再確認する

### 想定結果
- 追加したスポットがページ更新後も一覧に残る
- ブラウザを再起動しても同じ URL で再表示できる

### 合否判定
- リロード後にスポットが消える場合は `FAIL`
- 残っていれば `PASS`

---

## 9. PWA / マニフェスト確認
### 手順
1. ブラウザのデベロッパーツールから `Application` / `Manifest` を開く
2. `manifest.json` が読み込まれているか確認する
3. モバイルであれば「ホーム画面に追加」操作が可能か確認する

### 想定結果
- `manifest.json` が `app/layout.tsx` で読み込まれ、アプリ名・アイコンなどが表示される
- 本番ビルド環境ではサービスワーカーが登録される
- 開発環境では UI の確認のみ実施し、PWA 本番検証はビルド済みサイトで行う

### 合否判定
- マニフェストが未読み込み、またはホーム画面追加ができない場合は `FAIL`（ただし本番ビルド依存あり）
- 正常に読み込めれば `PASS`

---

## 総合判定
- すべての項目が `PASS` なら合格
- 1つでも `FAIL` があれば、問題箇所を特定して報告する

## 注意点
- `位置情報許可` と `拒否` は別々に確認すること
- `PWA` は開発モードでは完全に動作しない可能性があるため、本番ビルド検証を別途実施する
- `localStorage` は同一ブラウザ・同一ドメインでのみ保持される
